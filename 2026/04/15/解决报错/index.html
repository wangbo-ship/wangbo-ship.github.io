解决 Git 与 Hexo 的 400 连接报错

发表于2026-04-15|更新于2026-04-17|技术
|浏览量:

一、Git 报错：fatal: unable to access 'xxx': CONNECT tunnel failed, response 400

这个 400 错误，通常是由于你的 Git 配置文件（.gitconfig）中存在多重代理冲突导致的。

你可能在 C:/Users/Administrator/.gitconfig 中，同时配置了：

  1. 全局 HTTP/HTTPS 代理：指向 127.0.0.1:7890（本地代理工具，如 Clash）。
  2. 针对 GitHub 的特定代理：指向 https://ghproxy.net:443。
  3. 凭据提供程序：也指向了 ghproxy.net。

当你关闭本地代理后，Git 会尝试通过 ghproxy.net 镜像访问，但该镜像的隧道协议不匹配，从而导致 400 错误。

解决方案：彻底清理并重置代理

请在 Git Bash 或命令行中，依次执行以下命令来清理冲突的代理设置：

1. 移除所有冲突的代理设置

git config --global --unset http.proxy
git config --global --unset https.proxy
git config --global --unset http.https://github.com.proxy
git config --global --unset credential.https://ghproxy.net.provider
